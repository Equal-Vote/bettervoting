📚 How It Works

Users

  • Alice Owner - The only user who can create elections.
  • Bob Admin, Carol Admin - Will be admins once Alice adds them.
  • Dave Auditor, Eve Auditor - Can audit but not modify.
  • Voters (10) - Regular voters with unique device IDs.

Election Lifecycle

  1. No Election - Starting state. Only Alice can create.
  2. Draft - Test ballots allowed. Cleared on finalize.
  3. Finalized - Waiting for start time (if set).
  4. Open - Accepting real votes.
  5. Closed - No more votes. Results viewable.
  6. Archived - Read-only historical record.

Voter Authentication Types

  • Device ID - One vote per browser/device
  • Email - Requires login
  • IP Address - One vote per IP
  • None - Unlimited votes

Key Behaviors

  • Test ballots in draft mode are deleted when finalized
  • Only the election owner can change election state
  • Admins/Auditors get their roles after being added by owner
  • Each voter has a unique device ID for vote tracking
